- (decision) 데이터=Kaggle Lending Club accepted loans(2007~2018Q4). train/valid=2012-2014 빈티지, OOT=2015 빈티지로 시점 분리(진짜 OOT 확보). 36개월물 우선(만기 성숙도)
- (decision) 부도 라벨 정의: loan_status in (Charged Off, Default)=bad, Fully Paid=good, Current 등 진행중 건 제외
- (decision) 성과기간은 만기 기준 최종 상태로 정의(월별 연체이력 데이터 부재). 12개월 창 근사는 last_pymnt_d 기반 부록 실험으로 분리, MDD에 선택근거 기록
- (decision) 챔피언 모델=WOE 로지스틱 스코어카드(PDO=20, Base Score=600). 챌린저=LightGBM+Optuna 튜닝+isotonic/Platt calibration(챌린저 한정)
- (decision) 결측치는 WOE 별도 빈으로 처리, 대치(imputation)는 최소화 — WOE 비닝 표준 관행 우선
- (decision) IV 변수선정 후 상관계수/VIF로 다중공선성 중복 변수 제거 단계 추가
- (decision) reject inference(거절건 미관측 편향)는 공개데이터로 구현 불가 — MDD 한계 섹션에 실무 보정 방법을 서술로 대체, 코드 구현 범위 아님
- (decision) reason code는 모형 유형별 이원화: 챔피언=특성별 점수손실(points_lost), 챌린저=SHAP(shap_value) — 동일 응답 스키마, 값 필드만 상이
- (decision) 스코어링 API(/v1/score)는 승인/거절 판정을 내리지 않는다 — 점수·PD·등급·reason_codes만 반환, cutoff 적용은 소비자(Streamlit 대시보드, 추후 P3 loan-agent-lab 에이전트)의 책임으로 분리
- (decision) 컨설턴트 킥 4종 채택: 손익기반cutoff, 룰효율성진단, 비금융텍스트파생변수, SAS재현. 목적은 리스크 지표(KS/PSI/AUC)를 손익·룰 언어로 번역해 '모델러'가 아닌 '심사전략 컨설턴트' 포지셔닝 확보(채용공고 우대사항 대응)
- (decision) 손익기반cutoff(킥1)는 int_rate(수익)와 recoveries/total_pymnt(손실)로 건별 실현손익 산출, 결과 API 응답에 assumptions 필드로 가정을 항상 명시(실제 재무데이터 아님을 명확화)
- (decision) 룰효율성진단(킥2)의 verdict(유지/재검토 권장)는 배제집단 부도율 비율과 모형점수 상관도 기반 규칙 산출 — LLM 생성 아님. 결정적 판정 원칙을 P1에서도 유지
- (decision) 비금융 텍스트 파생변수(킥3)는 emp_title 필드 한정, 정제·상위빈도 카테고리 매핑 정도로 범위 최소화(정교한 NLP 지양). Home Credit 2차 데이터 활용은 이 모듈로 대체, 범위에서 제외
- (decision) SAS 재현(킥4)은 SAS OnDemand for Academics(무료)에서 스코어카드 점수산출 로직(WOE변환→선형결합→PDO스케일링) 1개만 이식, Python 산출값과 오차 0.5점 이내로 대조 검증
- (direction) P3(loan-agent-lab, 별도 프로젝트)가 향후 이 프로젝트의 /v1/score, /v1/grades를 도구로 소비할 예정 — API_SPEC.md 버저닝·연동 계약 섹션을 동결된 인터페이스로 취급, 본 SPEC의 범위에는 P3 자체 구현은 포함하지 않음
- (capability) CAP-1 표본 설계: 신청시점 기준 누수 필드를 배제하고 빈티지 기반 train/valid/OOT를 분리한다 (FR-1,2). success: 배제 필드 목록 문서 존재 + train/valid/OOT 건수·부도율 테이블 산출
- (capability) CAP-2 결측·이상치 전처리: 결측=별도 빈, 이상치=캡핑 규칙으로 정제한다 (FR-3). success: 정제 규칙 문서 + 정제 전후 분포 비교 리포트
- (capability) CAP-3 WOE 비닝 및 변수선정: 전 변수를 WOE/IV로 비닝하고 IV+상관/VIF로 중복을 제거해 최종 변수셋을 선정한다 (FR-4,5). success: IV 테이블 산출 + 최종 변수셋의 pairwise 상관 임계치(예: 0.7) 이하 충족
- (capability) CAP-4 로지스틱 스코어카드(챔피언): WOE 변수로 PDO=20, Base=600 스코어카드를 구축한다 (FR-6). success: 신청 1건 입력시 점수 산출 가능, 계수 전부 부호가 비즈니스 상식과 일치
- (capability) CAP-5 LightGBM 챌린저: Optuna로 튜닝한 LightGBM을 학습하고 isotonic/Platt calibration을 적용한다 (FR-7,9). success: calibration 전후 Brier score 개선 확인, 캘리브레이션 곡선 산출
- (capability) CAP-6 모형 평가: 챔피언·챌린저를 train/valid/OOT 3면에서 AUC·KS·PR-AUC로 비교한다 (FR-8). success: 3면 비교표 산출, OOT 챔피언 KS>=0.25 & 챌린저 AUC>=0.70
- (capability) CAP-7 등급 매핑: 점수를 1~10등급으로 매핑하고 등급별 부도율 단조성을 검증한다 (FR-10). success: 등급별 관측 부도율이 완전 단조 감소
- (capability) CAP-8 PSI 안정성 검증: train 빈티지 대비 OOT 빈티지의 변수·점수 분포 PSI를 산출한다 (FR-11). success: 점수 PSI < 0.1
- (capability) CAP-9 리스크 기반 Cutoff 시뮬레이션: cutoff 점수에 따른 승인율-부도율 트레이드오프 곡선을 산출한다 (FR-12). success: 전 구간 curve 데이터 + 특정 cutoff의 승인율/부도율 즉시 조회 가능
- (capability) CAP-10 Swap-set 분석: 챔피언과 챌린저 교체시 판정이 뒤바뀌는 고객군을 정량화한다 (FR-13). success: swap-in/swap-out 건수와 각 집단 부도율 비교표 산출
- (capability) CAP-11 Reason Code: 개별 신청 건의 점수 하락 사유 상위 3개를 모형 유형별(점수손실/SHAP)로 산출한다 (FR-14). success: 임의 1건에 대해 사유 3개 + 완성된 한국어 설명 문장 반환
- (capability) CAP-12 스코어링 API: FastAPI로 /v1/score, /v1/score/batch, /v1/model/info, /v1/grades, /v1/simulate/cutoff를 서빙한다 (FR-15, API_SPEC.md). success: 전 엔드포인트 pytest 통과 + /v1/score p95<300ms
- (capability) CAP-13 Streamlit 대시보드: 모형 성능·등급분포·PSI·cutoff 시뮬레이션을 시각화한다 (FR-16). success: 4개 화면 라이브 구동, 스크린샷 확보
- (capability) CAP-14 손익 기반 Cutoff(킥1): int_rate/recoveries로 건별 실현손익을 계산해 이익극대화 cutoff와 경영진용 1페이저를 산출한다 (FR-17). success: 현재 cutoff 대비 최적 cutoff의 승인율·연간기대손익 delta가 assumptions와 함께 산출
- (capability) CAP-15 룰 효율성 진단(킥2): 가상 하드룰셋을 적용해 룰별 배제집단 부도율·기회손실·유지/재검토 verdict를 산출한다 (FR-18). success: 룰 3개 이상에 대해 규칙기반 verdict가 포함된 리포트 산출
- (capability) CAP-16 비금융 텍스트 파생변수(킥3): emp_title을 정제해 파생변수화하고 IV 기여도를 검증한다 (FR-19). success: 파생변수의 IV 값과 정형 변수 대비 기여도 비교 결과 문서화(효과 유무 무관)
- (capability) CAP-17 SAS 재현(킥4): 스코어카드 점수산출 로직을 SAS로 이식해 Python 산출값과 대조 검증한다 (FR-20). success: 동일 입력 10건 이상에서 SAS-Python 점수 오차 < 0.5점
